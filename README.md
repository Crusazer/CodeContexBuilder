# Prompt Workshop (CodeContextBuilder)

> **Мощный десктопный ИИ-агент и среда для сборки промптов с применением изменений прямо в кодовую базу.**

Проект значительно эволюционировал: теперь это не просто утилита для копирования контекста, а **полноценная агентская система** с графическим интерфейсом на PyQt6. Инструмент позволяет не только формировать идеальные промпты из локальных файлов, но и автономно выполнять задачи, редактировать код через Diff-движок и следовать многошаговым воркфлоу.

---

## 📌 Основная идея

Вместо того чтобы вручную переносить файлы туда-сюда между IDE и LLM-чатами, вы можете доверить эту работу **Prompt Workshop**:
- Соберите точный контекст из нужных файлов (включая только что измененные в Git).
- Соберите промпт из готовых модулей (Роли, Правила, Навыки, Форматы).
- Поручите агенту написать код.
- Агент сам создаст файлы или вернет `SEARCH/REPLACE` блоки, которые приложение автоматически и безопасно (с созданием `.bak` копий) применит к вашему проекту.

---

## 🛠️ Новый и расширенный функционал

- 🤖 **Полноценная Агентская Система** — встроенный ИИ-цикл (Agent Loop) с поддержкой вызова инструментов (`Create File`, `Edit File`, `Reasoning`). Агент может автономно работать над задачей, запрашивая подтверждение у пользователя (Human-in-the-loop).
- 🔄 **Многошаговые Workflows** — движок для сложных задач, проводящий задачу через разные стадии: `Оркестратор` → `Архитектор` → `Разработчик` → `Ревьюер`.
- 🛠️ **Интеллектуальный Diff-движок** — автоматический парсинг ответов модели (формат `<<<<<<< SEARCH ... >>>>>>> REPLACE`) и безопасное применение патчей к файлам проекта с предварительным созданием бекапов.
- 🧩 **Модульный конструктор промптов** — управление шаблонами (Роли, Навыки, Правила, Форматы вывода) через удобный UI. Шаблоны хранятся в `.md` файлах с YAML frontmatter.
- 🐙 **Умная работа с файлами и Git** — выбор только измененных файлов одним кликом, поддержка `.gitignore`, умное игнорирование бинарных файлов.
- 💀 **Режим "Скелета" (AST Parsing)** — возможность извлечения только сигнатур классов, функций и докстрингов (для Python) для существенной экономии токенов. Выбор режима для каждого файла отдельно.
- 💾 **Воркспейсы (Workspaces)** — возможность сохранить текущий воркфлоу и выбранный контекст в рабочую область и вернуться к ней позже.
- 🎨 **Графический интерфейс на PyQt6** — темная и светлая темы с контрастными индикаторами, поддержка горячих клавиш, нативная работа на Windows, macOS и Linux.
- 📊 **Точный подсчет токенов** — встроенная интеграция с `tiktoken` (кодировка `cl100k_base`) для постоянного контроля размера контекста.

## 📦 Установка

1. Убедитесь, что у вас установлен Python 3.13+
2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yourusername/codecontextbuilder.git
   cd codecontextbuilder
   ```
3. Установите зависимости (рекомендуется использовать `uv`):
   ```bash
   uv venv
   source .venv/bin/activate  # Для Windows: .venv\Scripts\activate
   uv pip install -r requirements.txt
   ```
4. Запустите приложение:
   ```bash
   python -m src.main
   ```

*(Для использования AI-функций потребуется настроить API ключ OpenAI или совместимого провайдера в файле `settings.json` или в настройках интерфейса)*

## 🚀 Сценарии использования

### 1. Автономный рефакторинг или багфикс
Откройте проект, выделите измененные в Git файлы или нужные модули в дереве. Напишите задачу (например: *"Переведи этот модуль на асинхронный лад"*). Запустите агента. Агент проанализирует код, подумает (Reasoning) и напишет Diff-блоки для замены. Нажмите "Применить диффы" — и ваш код будет обновлен!

### 2. Подготовка идеального промпта для внешних LLM (Claude/ChatGPT)
Соберите нужные файлы. Для огромных файлов включите режим `Skeleton`. На панели Prompt Builder выберите Роль (например, *"Senior Python Developer"*), Навыки и Формат вывода (*"Diff-only"*). Нажмите "Скопировать промпт" — в вашем буфере обмена окажется готовый, структурированный Markdown, который поймет любая нейросеть.

### 3. Выполнение сложных задач через Воркфлоу
Откройте панель Workflow, выберите встроенный сценарий (например, `New Feature` или `Refactor`). Приложение проведет вас по шагам. Сначала ИИ-Оркестратор разобьет фичу на подзадачи, затем Архитектор продумает структуру классов, а Воркер напишет код шаг за шагом. Воркфлоу автоматически переключает шаблоны и настройки промпта на каждом этапе.

## 📂 Структура проекта

- `src/core/` — Логика приложения (DiffEngine, FsScanner, PromptBuilder, AI/Agent Service, WorkflowEngine).
- `src/ui/` — Графический интерфейс на PyQt6 (Панели, Диалоги, Темы).
- `src/models/` — Схемы данных (Pydantic).
- `src/config.py` — Конфигурация и работа с `settings.json`.
- `templates/` — Markdown-шаблоны ролей и правил с YAML frontmatter для менеджера шаблонов.
- `workspaces/` — Сохраненные сессии и воркфлоу.
