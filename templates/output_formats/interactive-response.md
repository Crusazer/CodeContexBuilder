# Формат интерактивного ответа

Ответ содержит одну из трёх структур — строго одну за раз.

## Вариант A: Запрос файлов (когда информации недостаточно)

```
📋 ANALYSIS
<2-5 предложений: что понял из AGENTS.md/DEPS.md и задачи>

🔧 TOOL: READ_FILE
📂 path: src/core/diff_engine.py
💡 reason: <почему нужен>

🔧 TOOL: READ_FILE
📂 path: src/controller.py
💡 reason: <почему нужен>

⏸️ Ожидаю содержимое запрошенных файлов для продолжения.
```

---

## Вариант B: План (когда информации достаточно для плана, но не для диффов)

```
📋 ANALYSIS
<анализ задачи и контекста>

📝 PLAN
1. <шаг 1 — что делаем>
2. <шаг 2 — что делаем>
3. <шаг 3 — что делаем>

📂 FILES AFFECTED
- `src/core/diff_engine.py` — <что меняется>
- `src/controller.py` — <что меняется>

⚠️ NOTES
- <важные замечания/риски>
```

---

## Вариант C: Диффы (когда информации достаточно для реализации)

Используй стандартный SEARCH/REPLACE:

## File: path/to/file.py
`````
<<<<<<< SEARCH
exact original code including whitespace
=======
new replacement code
>>>>>>> REPLACE
```

---

## Правила формата

1. Всегда начинай с `📋 ANALYSIS`.
2. Выбери ровно один вариант (A, B или C).
3. Не смешивай TOOL-запросы с диффами.
4. При диффах — каждый `SEARCH` должен встречаться в файле ровно 1 раз.
5. Пустой `SEARCH` = создание нового файла.
```
