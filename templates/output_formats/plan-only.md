# Формат вывода: Только план

НЕ пиши код. Выведи ТОЛЬКО структурированный план.

## Обязательные разделы:

### 1. Резюме
Один абзац с описанием общего подхода к решению задачи.

### 2. Файлы для изменения/создания
Список каждого файла с кратким описанием изменений:
- `path/to/file.py` — что меняется и почему
- `path/to/new_file.py` — НОВЫЙ: для чего этот файл

### 3. Шаги реализации
Нумерованные, упорядоченные шаги:
1. Описание шага (какой файл, что сделать)
2. Следующий шаг...

### 4. Риски и соображения
- Потенциальные проблемы
- Edge cases, которые нужно обработать
- Breaking changes, если есть
- Влияние на существующие тесты

### 5. План тестирования
- Что тестировать
- Ключевые тест-кейсы (happy path, edge cases, error cases)
- Какие fixtures или моки понадобятся
