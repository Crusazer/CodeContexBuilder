# Итеративный воркфлоу

## Порядок работы

### Раунд 1: Анализ и запрос
1. Проанализируй AGENTS.md + DEPS.md + задачу
2. Определи: достаточно ли информации?
3. Если нет → выведи `📋 ANALYSIS` + `🔧 TOOL`-запросы
4. Если да → переходи к финальному ответу

### Раунд 2: Дополнение (если были запросы)
1. Проанализируй полученные файлы
2. Если нужно → запроси ещё (не более 5 за раз)
3. Если достаточно → переходи к финальному ответу

### Раунд 3+: Финальный ответ
- Выдай результат в запрошенном формате (diff / plan)
- Больше не запрашивай файлы

## Жёсткие правила

1. **Не угадывай** — если не уверен в содержимом файла, запроси его.
2. **Не запрашивай лишнее** — если задача простая и контекст достаточен, отвечай сразу.
3. **Не смешивай** — либо запрашиваешь файлы (🔧 TOOL), либо отвечаешь (diff/plan). Не одновременно.
4. **Максимум 2 раунда запросов** — потом отвечай на основе имеющегося.
5. **Всегда начинай с `📋 ANALYSIS`** — краткий разбор что понял из контекста.
6. **Если запрашиваешь файлы** — заканчивай ответ строкой: `⏸️ Ожидаю содержимое запрошенных файлов для продолжения.`
