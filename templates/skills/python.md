# Навык: Python

## Версия и возможности:
- Python 3.12+ (type hints, `match-case`, `StrEnum`, `ExceptionGroup`, `TaskGroup`)
- Используй современный синтаксис: `list[str]` вместо `List[str]`, `X | None` вместо `Optional[X]`

## Типизация:
- Аннотации типов для ВСЕХ сигнатур функций (аргументы + возврат)
- `from __future__ import annotations` для отложенной интерпретации (forward references)
- `TypeAlias` для сложных составных типов
- `Protocol` вместо ABC там, где нужна структурная типизация
- `TypeVar` и `Generic` для обобщённых контейнеров/сервисов
- Для словарей с фиксированной структурой — `TypedDict`

## Стиль кода:
- Docstrings для публичных функций и классов (Google-стиль)
- f-строки для форматирования, не `.format()` и не `%`
- `pathlib.Path` вместо `os.path`
- `dataclasses` для простых структур данных, `pydantic.BaseModel` для валидации
- Контекстные менеджеры (`with`) для ресурсов (файлы, соединения, блокировки)
- Генераторы списков/словарей, если они читаемы (не вкладывай больше двух уровней)
- Walrus-оператор (`:=`) когда он действительно упрощает код
- `Enum` / `StrEnum` для фиксированных множеств значений, не строковые константы

## Обработка ошибок:
- Конкретные исключения, никогда голый `except:` или `except Exception:`
- Кастомные исключения для доменных ошибок, наследуемые от общего базового
- Модуль `logging` с именованными логгерами (`logger = logging.getLogger(__name__)`), не `print()`
- `contextlib.suppress()` вместо пустого `except ... : pass`
- Цепочки исключений: `raise NewError(...) from original_error`

## Импорты:
- Порядок: стандартная библиотека → сторонние пакеты → локальные (разделены пустой строкой)
- Абсолютные импорты предпочтительнее относительных
- Не используй `from module import *`
- Группируй связанные импорты из одного пакета

## Структуры данных и паттерны:
- `collections.defaultdict`, `Counter`, `deque` — используй стандартную библиотеку
- `functools.lru_cache` / `@cache` для мемоизации чистых функций
- `itertools` для эффективной работы с итераторами
- `contextlib.contextmanager` для кастомных контекстных менеджеров
- `__slots__` для классов с большим числом экземпляров

## Асинхронность:
- `asyncio` для I/O-bound задач
- `async with` / `async for` для асинхронных ресурсов и итераторов
- `asyncio.TaskGroup` (Python 3.11+) вместо `gather` для структурированной конкурентности
- Не смешивай sync и async код без явного выноса в `run_in_executor`

## Тестирование:
- `pytest` с фикстурами, параметризацией (`@pytest.mark.parametrize`)
- Осмысленные имена тестов: `test_create_user_with_duplicate_email_raises_conflict`
- `pytest-asyncio` для асинхронных тестов
- Фикстуры с `yield` для setup/teardown
- `unittest.mock.patch` / `MagicMock` — мокай внешние зависимости, не внутреннюю логику
- Отдельные `conftest.py` для разных уровней тестов
