# Правило: сохраняй существующий код

При модификации файлов ОБЯЗАТЕЛЬНО сохраняй всю существующую функциональность, не связанную с текущей задачей.

## Правила:
1. **Не удаляй** существующие функции, методы или классы, если это не запрошено явно
2. **Не переименовывай** публичные API (имена функций, классов, методов, аргументов)
3. **Не меняй** сигнатуры существующих функций (новые параметры — только необязательные, в конец)
4. **Не удаляй** существующие импорты, если они реально используются
5. **Не модифицируй** существующие тесты, если они не тестируют изменённое поведение
6. **Не реорганизовывай** структуру кода, если это НЕ является задачей
7. **Сохраняй** все существующие комментарии и docstrings
8. **Не меняй** форматирование в строках, которые не затрагиваются изменениями

## При использовании SEARCH/REPLACE:
- Блок SEARCH должен ТОЧНО совпадать с существующим кодом — включая отступы, пробелы, переносы строк
- Включай только минимальный контекст, достаточный для уникальной идентификации места
- Вноси точечные, хирургические изменения — не перезаписывай большие блоки ради одной строки

## Принцип минимального воздействия:
- Если нужно добавить метод в класс — добавь только метод, не перезаписывай весь класс
- Если нужно изменить одну строку — меняй одну строку, не весь файл
- Если нужно добавить импорт — добавь его в правильное место, не переписывай блок импортов
