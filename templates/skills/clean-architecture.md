# Навык: Чистая Архитектура (Clean Architecture)

## Структура слоев:
1. **Entities (Сущности)** — ядро бизнес-логики, классы без внешних зависимостей. Включает Value Objects (объекты-значения).
2. **Use Cases (Сценарии использования / Интеракторы)** — специфичные для приложения бизнес-правила и оркестрация логики.
3. **Adapters (Адаптеры / Порты)** — преобразование данных между слоями (контроллеры, презентеры, шлюзы, репозитории).
4. **Infrastructure (Инфраструктура)** — фреймворки, БД, брокеры сообщений, внешние API.

## Правило зависимостей (Dependency Rule):
- Зависимости направлены ТОЛЬКО ВНУТРЬ (от инфраструктуры к сущностям).
- Внутренние слои ничего не знают о внешних (никаких импортов ORM или FastAPI в бизнес-логике).
- Используйте интерфейсы/протоколы (в Python: `abc.ABC` или `typing.Protocol`) на границах слоев для инверсии зависимостей.

## Принципы SOLID:
- **S**ingle Responsibility (Единственная ответственность) — у модуля/класса должна быть только одна причина для изменения.
- **O**pen/Closed (Открытость/Закрытость) — программные сущности открыты для расширения, но закрыты для модификации.
- **L**iskov Substitution (Подстановка Барбары Лисков) — объекты базового класса можно заменить объектами подклассов без поломки логики.
- **I**nterface Segregation (Разделение интерфейса) — много мелких, специализированных интерфейсов лучше одного универсального (`Protocol` в Python отлично для этого подходит).
- **D**ependency Inversion (Инверсия зависимостей) — модули верхних уровней не зависят от модулей нижних уровней. Оба зависят от абстракций.

## Практические правила для Python:
- **Никаких ORM-моделей в бизнес-логике**. Используйте DTO или чистые dataclasses/pydantic-модели.
- Используйте **Паттерн Репозиторий (Repository)** для абстрагирования доступа к данным.
- Используйте **Паттерн Единица работы (Unit of Work)** для управления транзакциями.
- Используйте **Паттерн Внедрение зависимостей (Dependency Injection)**. В Python для этого можно использовать DI-контейнеры (например, `dishka` или `dependency-injector`) или фичи фреймворка.
- Взаимодействие между доменами предпочтительно строить через **Domain Events (Доменные события)**.
