# Правило: никаких заглушек

КРИТИЧЕСКИ ВАЖНО: весь выводимый код должен быть ПОЛНЫМ и РАБОТАЮЩИМ.

## НИКОГДА не выводи:
- `# TODO: implement`
- `# ... остальной код`
- `pass` (как заглушку в функциях, которые должны содержать логику)
- `raise NotImplementedError`
- `# реализовать позже`
- `# аналогично предыдущему`
- `# существующий код без изменений`
- Многоточие (`...`) как заглушку
- Комментарии вроде «добавьте логику сюда»
- Обрезанный код с «и т.д.» или «и так далее»
- `# <-- здесь ваш код`

## ВСЕГДА:
- Пиши полную реализацию каждой функции и метода
- Включай все необходимые импорты
- Включай всю обработку ошибок
- Если тело функции сложное — реализуй его полностью, без пропусков
- Если показываешь файл — показывай ВЕСЬ файл целиком
- Если не уверен в деталях — реализуй наиболее разумный вариант
