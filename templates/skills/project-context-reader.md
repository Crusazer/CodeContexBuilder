# Project Context Reader

## Анализ проекта

Когда тебе предоставлены AGENTS.md и/или DEPS.md, используй их системно:

### Шаг 1: Архитектура (из AGENTS.md)
- Выдели слои и их ответственности
- Определи ключевые модули и их роли
- Зафиксируй конвенции кода и ограничения
- Найди «подводные камни» и точки расширения

### Шаг 2: Зависимости (из DEPS.md)
- Построй граф зависимостей в уме
- Найди модули, связанные с задачей
- Обрати внимание на ⚠️-маркеры (неразрешённые символы)

### Шаг 3: Соотнесение с задачей
- Какие модули затронуты задачей?
- Какие файлы из этих модулей нужно прочитать?
- Есть ли зависимости, которые тоже нужно проверить?

## Эвристики для поиска файлов

| Тип задачи | Что запрашивать |
|---|---|
| Баг в модуле | Файл модуля + его зависимости из DEPS.md |
| Новый функционал | Файл куда добавляется + смежные модули + модели |
| Рефакторинг | Модуль + все зависимые от него (обратные deps) |
| Новая панель UI | Существующая панель как пример + controller + models |
| Изменение формата | diff_engine.py + всё что использует формат |

## Формат анализа

Перед запросом файлов кратко опиши:
1. Что ты понял из AGENTS.md — архитектура, ключевые модули
2. Что ты понял из DEPS.md — граф зависимостей, релевантные связи
3. Чего не хватает — конкретные файлы и причины
