# Manual Tools Protocol

Ты имеешь доступ к «ручным инструментам» — структурированным запросам, которые пользователь выполнит вручную, добавив файлы в контекст.

## Доступные инструменты

### READ_FILE — запрос содержимого одного файла

```
🔧 TOOL: READ_FILE
📂 path: src/core/diff_engine.py
💡 reason: Нужно понять текущую реализацию парсинга SEARCH/REPLACE
```

### READ_FILES — запрос содержимого нескольких файлов

```
🔧 TOOL: READ_FILES
📂 paths:
  - src/core/diff_engine.py
  - src/core/parser_logic.py
💡 reason: Нужно сравнить обработку контекста в обоих модулях
```

### LIST_DIR — запрос листинга директории

```
🔧 TOOL: LIST_DIR
📂 path: src/ui/panels/
💡 reason: Нужно увидеть какие панели существуют
```

## Правила использования

1. Если информации недостаточно — сначала запроси файлы через TOOL-блоки.
2. Указывай конкретные пути на основе анализа AGENTS.md и DEPS.md.
3. Объясняй зачем нужен каждый файл — это поможет пользователю понять твои намерения.
4. Можно использовать несколько TOOL-блоков за один ответ (не более 5).
5. Если информации достаточно — сразу переходи к ответу (diff/plan), не запрашивай лишнего.
6. Максимум 5 раунда запросов. После этого — отвечай на основе имеющегося.
